# 배경 아카이브: 하네스 설계 리서치

이 문서는 GHCP Harness를 설계할 때 정리한 **문제 공간과 설계 패턴**을 기록한다. 공개 사용자에게 필요한 기능 설명은 [00-harness-features.md](00-harness-features.md), 실행 방법은 [08-run-harness.md](08-run-harness.md)를 먼저 본다.

## 1. 하네스란 무엇인가

여기서 하네스는 모델 자체가 아니라, 모델이 실제 코딩 작업을 수행하도록 둘러싼 **작업 구조**를 뜻한다.

| 구성 | 의미 |
| --- | --- |
| 역할/모드 | 탐색, 구현, 질문, 검증 같은 작업 단계를 나누는 방식 |
| 도구 권한 | 읽기, 검색, 편집, 실행을 언제 허용할지 정하는 방식 |
| 컨텍스트 관리 | 큰 코드베이스에서 필요한 정보만 유지하는 방식 |
| 검증 루프 | 수정 후 테스트·진단으로 완료를 판정하는 방식 |
| 안전장치 | 보호 경로, 승인 게이트, 비가역 작업 제한 |
| 운영 기록 | 결정 로그, 보류 백로그, 상태 정본 |

## 2. 참고한 설계 패턴

GHCP/VS Code가 이미 제공하는 커스터마이징 표면으로 옮길 수 있는 패턴을 정리했다.

| 패턴 | 이 하네스에서의 표현 |
| --- | --- |
| 탐색과 구현 분리 | Plan / Build 에이전트 분리 |
| 최소 도구 권한 | `tools` allowlist와 read-only Explore |
| 프로젝트 규칙 파일 | 단일 [.github/copilot-instructions.md](../.github/copilot-instructions.md) |
| 경로별 지침 | [.github/instructions/](../.github/instructions/)의 `applyTo` |
| 온디맨드 절차 | [.github/skills/](../.github/skills/) |
| 검증 기반 완료 | doctor, tests, smoke, `/finish` |
| 보호 경로 | Agent hooks + CI 보완 |

## 3. 의도적으로 가져오지 않은 것

| 제외한 방향 | 이유 |
| --- | --- |
| 자체 에이전트 런타임 | 이 프로젝트의 목표는 VS Code Copilot 위의 정책 레이어다. |
| 샌드박스 실행 플랫폼 | 현재는 비가역 작업 승인과 검증 루프로 충분하다. 반복 마찰이 관찰되면 재검토한다. |
| 상시 멀티에이전트 코딩 | 코드 작성의 책임과 컨텍스트 연속성을 흐릴 수 있어 read-only 탐색으로 제한한다. |
| 점수/순위 중심 운영 | 이 저장소의 운영 기준은 현재 기능, 검증 결과, 반복 관찰된 마찰이다. |

## 4. 이 문서의 상태

이 문서는 배경 아카이브다. 새로운 기능을 추가할 때의 채택 여부는 [06-harness-operating-plan.md](06-harness-operating-plan.md)의 "관찰된 실패" 원칙을 따른다.
