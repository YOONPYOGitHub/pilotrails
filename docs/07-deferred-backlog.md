# 07 — 보류 백로그 (Deferred Backlog)

> 이 문서는 **지금은 도입하지 않기로 한 확장 후보**와 각각의 **도입 트리거**를 기록한다. 목적은 같은 논쟁의 재발을 막고, "언제 다시 꺼낼지"를 미리 정해 두는 것이다.
> 원칙: 이 하네스는 **관찰된 실패에만 장치를 추가**한다([06](06-harness-operating-plan.md) 원칙 1·2). 아래 항목은 "나쁜 아이디어"가 아니라 **아직 트리거(반복 관찰된 실패)가 충족되지 않은 것**이다.
> 출처: 외부 deep-research 보고서 검토(2026-06-19). 보고서의 외부 비교·Star 수치는 이 환경에서 검증 불가하여 가중치를 낮췄고, 저장소 내부에서 검증된 사실에 무게를 뒀다.

## 분류 기준

| 구분 | 의미 |
| --- | --- |
| **보류(대기)** | 아이디어는 타당하나 실제 마찰이 아직 관찰되지 않음. 트리거 충족 시 재검토. |
| **설계상 비채택** | 이미 근거를 갖고 안 하기로 결정함. 트렌드만으로는 번복하지 않음. |

## 보류(대기) 항목

| # | 항목 | 왜 보류 | 도입 트리거 |
| --- | --- | --- | --- |
| D1 | **`harness init` / `pack` 도입 CLI** | 복사형 도입의 마찰이 아직 관찰된 적 없음. CLI 자체가 유지보수 대상(버전·머지·전제조건 체크)이 되어 과설계 위험. | 복사형 도입을 여러 번 dogfood해 **기존 `.github/` 충돌·누락 파일** 같은 마찰이 반복 관찰되면 최소 형태로 도입. |
| D2 | **MCP / Playwright 통합 지점** | README가 명시한 non-goal(자체 런타임·실행환경 아님)과 충돌. 현재 작업에서 MCP·브라우저 자동화가 필요했던 적 없음. | 이 저장소의 실제 작업에서 외부 도구 연동이 **반복적으로 필요**해질 때, 정체성(정책 레이어)을 깨지 않는 선에서 옵션으로. |
| D3 | **텔레메트리 / 구조화 run log** | 추적할 운영 이력이 아직 적음. 먼저 깔면 측정이 목표가 되는 Goodhart 함정 위험([06](06-harness-operating-plan.md) 원칙 1). | dogfood 세션이 쌓여 **drift·비용·시간 추세를 비교할 데이터**가 충분해지면. |
| D4 | **플러그인 / manifest 시스템** | 현재 agents·hooks·skills는 수가 적어 수작업 배선이 안 깨짐. `harness-doctor`의 죽은 자산 감지가 같은 위험을 더 가볍게 막음. | 자산 수가 늘어 **수작업 배선이 실제로 깨지기 시작**할 때. |
| D5 | **eval 매니페스트 실행기** | `examples/evals/`는 현재 로드맵 자리만 있음. 평가 대상(여러 구성)이 먼저, 실행기는 나중. | **평가할 하네스 변형이 여러 개** 쌓이고 재현 가능한 비교 필요가 생길 때. |
| D6 | **devcontainer / Docker 재현 환경** | Preview 의존 마찰이 반복 관찰된 적 없음. Node 18+면 `node` 스크립트는 그냥 동작. | 협업자가 환경차로 hook·검증이 **반복적으로** 막힐 때. |

## 설계상 비채택 항목

| # | 항목 | 근거 |
| --- | --- | --- |
| N1 | **멀티에이전트 비교 모드 / 코딩 중 모델 전환** | [03](03-synergy-conflict-design.md)에서 이미 비채택 결론(Cognition·Anthropic 근거). 코드 작성은 단일 스레드, 서브에이전트는 read-only 전용, 모델 라우팅은 작업 경계에서만. 트렌드만으로 번복하지 않음. |

## 검토 시 재평가할 신호

다음이 관찰되면 위 항목을 다시 연다.

- 외부 사용자/협업자의 **반복되는 동일 마찰**(이슈·PR에서 같은 문제가 N회)
- dogfood 세션에서 **수작업으로 반복 보정**하게 되는 단계
- 보호·검증 자산이 늘어 **수작업 배선·정합 유지가 비용**이 되는 시점

> 재평가 결과는 [05-decision-log.md](05-decision-log.md)에 채택/기각으로 옮겨 적는다.
